# 画面別改修方針（2025-10-03）

## S1: ShoppingListScreen
1. **トップバー**
   - 背景にブランドブルー `#1B70A0` を適用し、タイトル文字を `DM Sans / SemiBold` で `onPrimary` 白に変更。
   - 右側に設定アイコン（ブランド角丸背景のOutlined gear）を配置して将来のテーマ切替・設定導線を確保。
2. **FAB**
   - 背景色をアクセントレッド `#EA5F52`、アイコンを白塗りの `Icons.Add` に変更。
   - 影をElevation 6dpで強調。
3. **リストアイテム**
   - カード背景を `#F6FAFD`、角丸16dp、左側にブランドブルーのティアドロップアイコンバッジを配置。
   - 購入済みアイテムはテキストカラーを `onSurfaceVariant` から `#4F5D6A` へ、背景に薄い青 `#E2F1FA` を適用。
4. **権限プロンプト**
   - SecondaryContainerの代わりにグラデーション（`#1B7CB0 → #1B6491`）を使用、白文字＋アイコンバッジで訴求。
5. **空状態**
   - 深い青背景＋白リストアイコンの簡易イラストを表示。文言にアクセントラインを追加。

## S2: ItemDetailRoute
1. **トップバー**
   - S1と同じブランドトップバーを再利用し、`ArrowBack` アイコンは角丸背景つきの白アイコンに統一。
2. **編集フォーム**
   - OutlinedTextFieldのフォーカス色をブランドブルーに変更し、プレースホルダー/ラベルを DM Sans Medium 16sp で統一。
   - 保存ステータス表示にアクセントレッドではなくブランドブルーのローディングリングを使用。
3. **場所リスト**
   - 各地点カードを角丸16dp＋陰影2dp、左帯にピンアイコンを配置。
   - 削除ボタンをアクセントレッド背景の丸ボタンに変更し、ツールチップとして`削除`を表示。
4. **追加ダイアログ**
   - ボタンカラーをPrimary/Accentに変更し、アイコン＋文言の二段レイアウトで視認性を高める。

## S3: PlacePickerScreen
1. **ヘッダー**
   - グラデーション背景＋透明ステータスバーで視覚的な広がりを作る。
   - Closeボタンを白背景の角丸アイコンに差し替え。
2. **検索欄**
   - 角丸24dpのフィールドにブランドブルーアウトライン、左側にブランドカラーの検索アイコンを配置。
3. **候補リスト**
   - リスト要素にアイコンバッジ＋サブテキスト（住所）のフォーマットを適用。
4. **地図**
   - 選択ピンをブランドカラーのカスタムマーカーに変更（赤ピン＋白アイコン）。
   - 地図最下部にCTAパネル（選択内容の概要と確定ボタン）を設け、アクセントレッドで強調。
5. **許可誘導**
   - 位置情報未許可時のカードをブランド青背景＋白文字で差し込み、権限取得を明示。

## S4: RecentPlacesRoute
1. **トップバー/背景**
   - 一覧を淡い青 `#F0F7FC` の背景に変更し、カードを浮かせる。
2. **リスト行**
   - 角丸16dpカード＋左側にブランドブルーの小バッジを配置。
   - 距離/更新日などのメタ情報を本文下に追加し、Captionスタイルを活用。
3. **空状態/エラー**
   - アイコンセット由来のピン＋顔アイコンを使ったイラストを表示し、行動導線（「お店を登録」）ボタンを付与。

## 共通パターン（S1〜S4）
- `Snackbar` はブランドブルー背景＋白文字で統一。
- `ButtonDefaults` を拡張し、Primary=ブルー、Secondary=白背景＋ブルーボーダー、Danger=アクセントレッドに設定。
- アイコン背景は 36dp 角丸矩形で統一し、`Icons.Default` の線幅を揃える。

## 実装優先度案
1. テーマ適用（共通色/フォント）
2. トップバー＆FAB改修（S1、S2）
3. リストカード改修（S1、S2、S4）
4. PlacePickerの検索/地図UI調整
5. 空状態・権限導線のイラスト追加
