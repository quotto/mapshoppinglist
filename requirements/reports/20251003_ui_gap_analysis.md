# UI現状把握レポート（2025-10-03）

## 共通テーマ/タイポグラフィ
- `Theme.kt` は `MaterialTheme` のデフォルト配色 (`lightColorScheme()`, `darkColorScheme()`) をそのまま使用。
- `Type.kt` もデフォルト `Typography()` を利用しており、DM Sans 等のブランドフォント設定なし。
- `colors.xml` は黒/白のみ定義。ブランドカラーやシステムカラーが未設定。
- すべてのトップバー/ボタン/背景がMaterial3既定（サーフェスや一次色）で描画され、アイコンセットの深い青＋赤のトーンと乖離。

## 画面別状況
### S1: ShoppingListScreen (`ShoppingListScreen.kt`)
- FABと削除ボタンは `Icons.Filled.Add/Delete` のマテリアル標準アイコン。ブランドカラー適用なし。
- コンポーネントはフラットなカード＋チェックボックスの構成で、角丸や配色がアイコンのラウンド感と一致していない。
- 空状態表示にブランドイメージ（ピン、青系背景等）が反映されていない。
- 権限プロンプトカードは `secondaryContainer` の淡色背景だがブランドカラーとの整合性が不明確。

### S2: ItemDetailRoute (`ItemDetailRoute.kt`)
- トップバー・ボタン・テキストフィールドすべてが標準配色。ブランド青を基調とした強調やアクセントの活用なし。
- アイコン/ボタンの角丸設定はデフォルト（4dp〜8dp）で、アイコンセットのティアドロップ型・大きめ角丸との統一が必要。
- リスト内の削除アイコン `Icons.Default.Delete` の赤はテーマ依存（`onSurfaceVariant`）。ピンレッド `#EA5F52` を行動色に置き換える余地。

### S3: PlacePickerScreen (`PlacePickerScreen.kt`)
- トップバーおよび検索欄はスタンダードなライトテーマ。背景が白基調でブランドブルーが存在しない。
- 進捗表示（Linear/CircularProgressIndicator）もデフォルトカラー。
- 地図UI上のマーカー、ハイライト色はGoogle標準。ブランドピンを差し替える検討が必要。

### S4: RecentPlacesRoute (`RecentPlacesRoute.kt`)
- 全画面が白背景＋デフォルトトップバー構成。ブランド感がない。
- 空状態/エラー表示が文字のみで、アクセント色やピンモチーフが活用されていない。

## 主要ギャップまとめ
1. **ブランドカラー未適用**: どの画面も深い青・赤のキーカラーを使用していない。
2. **フォント不統一**: DM Sans未導入。デフォルトRoboto系のまま。
3. **アイコンスタイル乖離**: マテリアル標準アイコンをそのまま使用し、ブランドピンのスタイルに合わせた再設計が必要。
4. **角丸・シャドウ規格欠如**: アイコンセットのラウンド感/シャドウ表現がUIコンポーネントに反映されていない。
5. **イラスト/空状態演出不足**: 空リストや権限プロンプトなどブランドトーンを感じさせるビジュアルが不足。

## 改善に向けた着眼点
- `colors.xml` と `Theme.kt` にブランドカラーパレット（ライト/ダーク）を定義し、コンポーネントに適用。
- `Type.kt` に DM Sans の `FontFamily` を設定し、見出し/本文のスタイルガイドを決める。
- 主要アクション（FAB、重要ボタン）にアクセントレッドを採用し、背景やカードに深い青のバリエーションを注入。
- アイコンは単色塗り＋角丸背景（例: 12dp〜16dp）でブランドカラーを適用したオリジナルアセットに置き換える。
- 空状態や権限カードにブランドカラーのイラスト/簡易図形（ピンやリストのモチーフ）を追加。
